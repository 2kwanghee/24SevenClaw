## 배경

현재 파이프라인은 `ANTHROPIC_API_KEY`를 unset하고 [claude.ai](<http://claude.ai>) 구독 시트로 실행한다. 하이브리드의 "클라우드 팔"은 조직 API 키로 컨테이너 안에서 헤드리스 실행이 가능해야 성립한다.

## 작업

* Docker 컨테이너 안에서 **조직 API 키(Anthropic API 또는 Bedrock)**로 `claude -p` 헤드리스 실행이 되는지 PoC
* 실제 **구독 시트 수** / **API TPM·RPM 상한** 실측·문서화
* 데스크탑 러너(구독 시트)와 클라우드 컨테이너(API 키)의 동시성·비용 특성 비교

## 완료 기준 (산출물 = 결정 메모)

- 컨테이너 조직-키 헤드리스 실행 가능 여부 (가능/불가/조건부)
- 동시성 상한 실수치 (시트 수, API TPM/RPM)
- **데스크탑 러너 앞당김 판단** (P2 vs P3 배치) — 구독 시트가 초기 동시성의 몇 %를 담당하나
