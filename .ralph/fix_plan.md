## P1: 기능 요구사항

- [ ] **[P0-2][spike] 구독 시트 기반 동시성 산정 (데스크탑 러너 주력)**
  > 요청사항: ## Phase 0 — 토대 · de-risking (SPIKE · 2026-07-15 재정의)

## 목표

실행을 **[claude.ai](<http://claude.ai>) 구독 OAuth 세션(구독 시트)** 주력으로 확정하고, 그 위에서 동시에 몇 개의 AI 개발 세션을 안정적으로 돌릴 수 있는지 산정한다. 조직 API 키(종량제 크레딧) 경로는 **최소 폴백으로 격하**.

## 재정의 사유

크레딧(종량제 API) 사용을 지양하고 예측 가능한 구독 시트로 개발한다는 방침 확정 — 고정가 SI 딜리버리 경제성에 맞음. 따라서 하이브리드의 무게추를 데스크탑 러너(구독 시트)로 옮긴다. 기존 조직-API-키 컨테이너 PoC 결과(인증 경로는 유효, 크레딧 부족으로 실응답 미검증)는 `docs/hybrid-runner-headless-poc.md`에 보존 — 폴백 참고용.

## 작업

* 현재 확보된 [claude.ai](<http://claude.ai>) 구독 플랜/시트 수 확인 (Anthropic Console — 사람 액션)
* 구독 세션(`ANTHROPIC_API_KEY` unset)으로 동시 `claude -p` 헤드리스 세션을 N개 띄웠을 때 안정적으로 도는 상한 실측 (**크레딧 불필요 — 구독 세션으로 측정**)
* 사용량 창(usage window)·레이트 제약이 동시성에 주는 영향 파악
* "시트 N개 → 동시 러너 M개" 매핑 산정

## 완료 기준 (산출물 = 동시성 예산 메모)

- 구독 플랜/시트 수 확인
- 구독 세션당 안정적 동시 헤드리스 세션 수 실측
- 시트→동시 러너 매핑 + 20~50 프로젝트 목표 대비 필요 시트 수 산정
- 데스크탑 러너 주력 설계 확정 (조직 API 키 팔은 폴백으로 격하)

## 영향
