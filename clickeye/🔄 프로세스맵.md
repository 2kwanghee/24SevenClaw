## 2) PM 모델 라우팅 파이프라인
Opus는 계획/설계, Sonnet은 구현 — 토큰 비용 최적화.
```mermaid
flowchart TD
    REQ["👤 사용자 요청"] --> PM["🧠 PM Agent · Opus<br/>복잡도 ≥ 0.7 판단"]
    PM --> SPEC["📋 구현 스펙 생성"]
    PM -. "복잡도 ≥ 0.7 위임" .-> DT["🔬 deep-thinker · Opus<br/>트레이드오프 분석"]
    DT --> SPEC
    SPEC --> IMPL["⚙️ 구현 실행<br/>Sonnet + Haiku"]
```

## 3) 하네스 엔지니어링 (5단계)
AI 코드 작성을 통제해 환각/오류 사전 차단.
```mermaid
flowchart LR
    P["0. PM<br/>Opus"] --> R["1. Router<br/>Sonnet"]
    R --> C["2. Context<br/>Haiku"]
    C --> L["3. Loop<br/>Sonnet"]
    L --> WK["4. Worker<br/>역할별"]
    L -. "테스트 실패 시 수정<br/>(MAX 5회)" .-> L
```

## 4) 거버넌스 게이트 (자동 머지)
자율 파이프라인이 main 직접 머지 직전 단일 SSOT로 검증+위험분류.
```mermaid
flowchart TD
    PIPE["🔁 auto_dev_pipeline.sh"] --> GATE["🚦 pre_merge_gate.py<br/>(SSOT)"]
    GATE --> CD["contract-drift<br/>(차단)"]
    GATE --> TR["ticket-ref<br/>(차단)"]
    GATE --> PT["plan-trace<br/>(권고)"]
    GATE --> RD["위험 강등<br/>(HIGH → PR 경로)"]
    CD --> DEC{"통과?"}
    TR --> DEC
    PT --> DEC
    RD --> DEC
    DEC -- "예" --> MERGE["✅ main 직접 머지"]
    DEC -- "아니오 / HIGH" --> PR["🔀 기존 PR 경로로 강등"]
```
