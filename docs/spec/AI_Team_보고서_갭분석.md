## 상세 갭 분석

### GAP-1: 마스터 PM AI 오케스트레이터 (보고서 §2, §4)

**보고서 기획**:
- 작업 분해 및 배정
- 프롬프트 표준화
- 결과 통합 및 비교
- 충돌 해결
- 리스크 탐지
- 보고 자동화
- 버전 관리
- 단계 전이 판단

**현재 상태**:
- `harness-router` 스킬: 사용자 요청 의도 분석만 수행
- `harness-loop`: 코드 작성→테스트 반복 루프 (MAX 5회)
- 작업 분해, 팀 배정, 결과 통합, 충돌 해결 로직 없음

**개선 필요**:
- 오케스트레이터 서비스 신규 구현
- 작업 분해 알고리즘 (요청 → 서브태스크 분할)
- AI Team 배정 로직 (역할별 에이전트 매칭)
- 결과 통합 엔진 (다수 AI 출력 병합)

**영향 범위**: `24SevenClaw-api/app/services/`, `.claude/skills/`

---

### GAP-2: 산출물 상태 머신 (보고서 §5)

**보고서 기획**:
```
Draft → Reviewed → Revised → Approved → In Development → Validated → Released
```
- 메타정보: 작성 AI / 리뷰 AI / 버전 / 승인 상태 / 변경 이력

**현재 상태**:
- Project 모델에 기본 CRUD만 존재
- 상태 필드 없음 (프로젝트 상태만 Linear로 관리)
- 산출물별 메타정보 추적 없음

**개선 필요**:
- Artifact 모델 또는 ProjectConfig 확장
- 상태 전이 엔진 (FSM)
- 전이 규칙 (예: Reviewed→Revised는 리뷰 완료 시에만)
- 메타정보 자동 기록 (작성 AI, 리뷰 AI, 타임스탬프)

**영향 범위**: `24SevenClaw-api/app/models/`, `24SevenClaw-contracts/protocol/`

---

### GAP-3: 교차 리뷰 파이프라인 (보고서 §3)

**보고서 기획**:
- 메인 AI: 초안 작성 / 방향 설정
- 서브 AI: 검토 / 보완 / 반론 / 대안
- 10단계 중 04(초안)→05(교차리뷰)→06(수정통합) 자동화

**현재 상태**:
- `harness-worker` 스킬에 WRITE_CODE / CODE_REVIEW 역할 정의
- 실제 실행 파이프라인 없음 (역할 분리만 개념적 정의)

**개선 필요**:
- 메인 AI 초안 생성 트리거
- 서브 AI 자동 교차 리뷰 호출
- 리뷰 결과 diff 생성 + 병합 로직
- 반론/대안 제시 프롬프트 체계

**영향 범위**: `24SevenClaw-api/app/services/`, pipeline 설정

---

### GAP-4: 품질 검증 게이트 (보고서 §3 Step 07)

**보고서 기획**:
- PM AI가 품질 검증 수행
- 통과 시 승인 → 다음 단계 전이
- 실패 시 수정 요청 → 재작업

**현재 상태**:
- lint/typecheck 수준의 정적 분석만
- 자동 품질 검증 파이프라인 없음
- 검증 결과 기반 상태 전이 없음

**개선 필요**:
- 품질 메트릭 수집 (코드 품질, 보안, 성능)
- 검증 통과/실패 기준 정의
- 상태 머신과 연동 (검증 실패 → Revised로 롤백)
- QA 에이전트 카탈로그 확장

**영향 범위**: `24SevenClaw-api/app/services/`, QA 에이전트 설정

---

### GAP-5: 보고 자동화 + 대시보드 (보고서 §4)

**보고서 기획**:
- 보고 자동화 (PM AI 핵심 기능 중 하나)
- 단계별 진행 상황 자동 리포팅

**현재 상태**:
- `/log-work` 스킬로 Linear에 수동 로그
- 대시보드 페이지 존재하나 시각화 컴포넌트 미구현
- 단계별 자동 보고서 생성 없음

**개선 필요**:
- 리포팅 API 엔드포인트
- 대시보드 시각화 (상태별 카운트, 타임라인, 품질 메트릭 차트)
- Linear 데이터 연동 표시

**영향 범위**: `24SevenClaw-web/src/app/dashboard/`, `24SevenClaw-api/app/api/v1/`

---

### GAP-6: KPI 정량화 (보고서 §6, Appendix)

**보고서 기획**:
- 업무 속도 향상
- 품질 향상
- 피로도 감소
- 의사결정 효율 개선
- 운영 표준화

**현재 상태**:
- KPI 측정 체계 전무
- 메트릭 수집 포인트 없음

**개선 필요**:
- KPI 프레임워크 설계 문서
- 측정 지표 정의 (처리 시간, 처리량, 결함율, 리뷰 통과율, AI 활용률)
- 수집 포인트 설계 (API 미들웨어, 에이전트 리포터)
- 대시보드 연동 준비

**영향 범위**: `docs/`, API 설계 문서, 향후 대시보드
