단순히 해당 내용을 프롬프트에 입력하는 즉시 실행하는 구조가 아니라 Linear에 등록해 상세 Plan을 계획한다.
절대 개발을 선행하지말고 Linear에 상세 플랜을 계획하고 계획된 플랜은 나의 컨펌을 통해서 작업을 진행한다.
아래 내용에 대해 새로운 개발 플랜에 대한 md 파일을 생성해 나에게 보고 할 수 있도록 한다.
각자 모듈별로 필요한 agent를 생성해 병렬 작업이 가능한지 판단하고 병렬 작업이 필요하면 병렬작업을 진행할수 있는 
가이드 지침 반드시 Linear에 기재할 수 있도록 한다.

   서비스를 통해 생성된 오토 파이프라인을 임의로 변경, 추적, 또는 커스텀 하는것을 방지하기 위해(단, 커스텀 에이전트, 스킬등 원하는 부분은 추가가 가능)
4. 해당 서비스 또는 개발에 대한 성숙도 평가지표를 시작시점에 또는 회원가입 시점에 추가해서 성숙도에 따라 1.번 기능처럼 기본 프리셋을 제공할지 아니면 조금 세부화 하는 프리셋을 제공할지를 결정 할수 있는 기능
5. 24Seven에 대한 키벨류가 없음. 이 시스템을 사용함으로 어떤 장점이 있는지에 대한 밸류가 누락된 상태니 이를 서비스 이용 고객이 느낄수 있는 구조로 변경

: Executive Summary
  AI를 단순 활용 도구 수준을 넘어 역할 기반 팀 구조로 조직화하여 실제 업무 프로세스에 적용하는 AI Team 운영
  모델을 제안한다. AI를 역할별 팀으로 구성하고, 마스터 PM AI가 전체 흐름을 통제하며, 사람은 전략 및 의사결정에
  집중함으로써 속도·품질·관리 체계를 동시에 확보하는 것이 목표다.
1. 추진 배경 및 문제 인식
   · 단일 AI 중심 사용 → 결과 편향 및 검증 부족
   · 멀티 AI 수동 비교 → 높은 피로도 및 비효율
   · 산출물 일관성 부족 → 문서 정합성 저하
   · 버전 및 흐름 통제 미흡 → 운영 리스크 발생
   → 기존 방식은 도구 수준 활용에 머물며, 조직형 업무 운영 모델로 확장이 어렵다.
2. 제안 개요
   운영 구조 (3계층)
   · 사람 (총괄 책임자) — 목표 / 우선순위 / 전략 / 최종 승인
   · 마스터 PM AI (오케스트레이터) — 작업 분해 / 통합 / 리스크 관리 / 버전 관리
   · 분야별 AI Team (실행 조직) — 기획 / 분석 / 설계 / UX / 개발 / QA / 문서화
3. 팀 운영 방식
   메인 + 서브 구조
   · 메인 AI: 초안 작성 / 방향 설정
   · 서브 AI: 검토 / 보완 / 반론 / 대안
   10단계 업무 프로세스
   · 01. 요청 입력 (사람)
   · 02. 작업 분해 (PM AI)
   · 03. 팀 배정 (PM AI)
   · 04. 초안 생성 (메인 AI)
   · 05. 교차 리뷰 (서브 AI)
   · 06. 수정 및 통합 (메인 AI)
   · 07. 품질 검증 (PM AI)
   · 08. 승인 (사람/자동)
   · 09. 다음 단계 전이
   · 10. 버전 및 이력 관리
   프로젝트 수행 단계
   Phase 0 프로젝트 정의 → Phase 1 문제 정의·탐색 → Phase 2 컨셉 수립 → Phase 3 상세 기획 → Phase 4 개발
   준비 → Phase 5 구현·리뷰 → Phase 6 QA·검증 → Phase 7 MVP·베타
